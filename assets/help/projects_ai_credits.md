```markdown
# Projects AI: API Keys & Usage

Use this guide to configure Projects AI with your own provider account or local endpoint.

---

## How Access Works

1. **Your API key** — Add your own provider key in Settings -> AI.
2. **Local AI** — Connect Ollama or another OpenAI-compatible local endpoint for local-only runs.
3. **User-managed usage** — Jot does not run a built-in wallet for AI access.

---

## Usage in Plain English

- Usage scales with text size and model choice.
- Jot can estimate provider usage before running when configuration supports it.
- Billing and limits are managed by your provider or local infrastructure.

---

## Common Scenarios

- **No API key configured** -> Add one in Settings -> AI, or connect a local provider.
- **Using a local model** -> Ensure your local server is reachable from this device.
- **Large multi-page paste** -> Expect higher latency and provider usage.

---

## Tips to Reduce Usage

- Keep inputs concise and remove duplicated text before running.
- Run conversion once, then edit the generated project manually.
- Use lighter models in AI settings for lower-cost runs.
- Prefer local AI when privacy and control matter most.

---

## Quick Steps

1. Open Projects AI and paste your raw text.
2. Review estimated usage (if available).
3. Run conversion and review the structured output.

---

## Need Help?

- Open **Settings -> Help & Guides -> Projects AI: API Keys & Usage**
- Check **Settings -> AI** to verify provider configuration
```
